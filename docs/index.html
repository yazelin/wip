60 秒看懂 wip 怎麼用

為什麼

手上二十幾個 repo,哪個還沒推、哪個有改動沒存?一個一個 cd 進去 git status 很累,而且你跟 AI 助理看的還不是同一份狀態。

人看表格
跑 wip,終端一張表:每個 repo 的分支、最後 commit、dirty / unpushed、開著的 PR。最近動的排前面。
AI 讀 markdown
跑 wip --md,輸出 markdown 摘要,Claude Code 開場自動讀進去 —— 你跟 AI 看同一份真相。

安裝

兩種方式,都不用一分鐘。

下載 release(免裝 Rust) —— 到 Releases 抓對應 OS 的壓縮檔:

# Linux x86_64
tar xzf wip-*-linux-x86_64.tar.gz
install -m 755 wip ~/.local/bin/wip       # 確認 ~/.local/bin 在 PATH

# 指向你的 repo 清單
mkdir -p ~/.config/wip
cp repos.example.toml ~/.config/wip/repos.toml   # 編輯這份清單

Windows:解壓 wip-*-windows-x86_64.zip,把 wip.exe 放進 PATH。

從原始碼裝:

cargo install --git https://github.com/yazelin/wip
mkdir -p ~/.config/wip
cp repos.example.toml ~/.config/wip/repos.toml

指令

指令做什麼
wip終端表格,最近 commit 的 repo 排最前
wip --mdmarkdown 摘要(給 Claude Code / SessionStart hook 讀)
wip --json結構化輸出
wip --root ~/dir臨時掃某資料夾下的所有 git repo(無視設定清單)
wip --no-gh跳過 gh(免網路、更快;PR/issue 欄顯示 -)
wip next <repo> "…"把待辦寫進該 repo 的 NEXT.md
wip done <repo> 1把第 1 個未完成待辦打勾

讀懂那張表

$ wip
wip — 25 repos (recent first)

ai-chant-magic  main
  36 分鐘前  Merge pull request #74  [3 dirty]
catime  main
  3 天前  Update RSS feed  [2 unpushed]
  • 分支名:目前 checkout 的 branch。
  • [N dirty]:有 N 個改動還沒 commit。
  • [N unpushed]:commit 了但還沒 push。
  • 時間 + 訊息:最後一次 commit。
  • 底下若有 see::偵測到 ROADMAP.md / TODO.md 等規劃檔,列檔名當閱讀指路(不解析內容)。

Claude Code / AI 整合

讓 Claude Code(和 Codex)在每次開場自動載入你的跨 repo 狀態:

wip install-hook          # 裝 SessionStart hook 進 ~/.claude/settings.json 與 ~/.codex/hooks.json
wip install-hook --print  # 或只印出片段,自己手動加

裝好後,AI 每次開場就先讀 wip --md,知道你各專案做到哪,不用你再解釋一遍。NEXT.md 跟著 repo 走(git 版控),你的下一步在哪台機都在。

裝成 skill(Claude + Codex) —— hook 是被動推狀態;想讓 agent「隨時主動」跑 wip、用 next / done,再裝 skill(也補 hook 到不了的 Codex):

wip install-skill   # 寫 ~/.claude/skills/wip 與 ~/.codex/skills/wip

Codex 需手動一步:install-hook 後在互動 Codex 跑一次 /hooks 信任該 hook。

視覺儀表板 & 關係圖

終端表格不夠看時,兩個選配 HTML —— 直接開檔(file://,免伺服器)。

狀態儀表板:dashboard.py 跑 wip --json、把結果內嵌成單一自足的 ~/wip-dashboard.html,一 repo 一張卡、依新舊上色(綠 <7 天、黃 <30 天、灰更舊、紅=錯),dirty/unpushed/PR 徽章。資料內嵌,file:// 也能開。

python3 dashboard.py   # 產 ~/wip-dashboard.html(預設帶 --no-gh)

關係圖(DIY):不自動生(分開的 repo 很少共用碼,硬推的圖沒用)。要的話用一張拉 Mermaid CDN 的 HTML,手畫 flowchart —— 用 subgraph 分群,只畫你敢背書的連線(實線=真依賴、虛線=規劃)。

設定清單

~/.config/wip/repos.toml 是一份**手選**的追蹤清單 —— 保持在「正在飛的專案」。清單過期(漏了在做的、留著收工的)是它變沒用的頭號原因,記得偶爾整理。

repos = [
  "~/project-a",
  "~/work/project-b",
]

想看「我最近實際碰了啥」而不管清單,直接 wip --root ~。
